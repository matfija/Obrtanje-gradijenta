Ublažavanje pristrasnosti neuronskih mreža¶

Robusnost modela mainskog uenja ogleda se u priblino jednakom uspehu nad raznim podacima. Ukoliko to nije sluaj, model je pristrastan. Poseban problem sa pristrasnou imaju neuronske mree, budui da su vrlo fleksibilne i (pre)prilagodljive podacima nad kojima su obuene. U svesci je predstavljena upotreba obrtanja gradijenta, kao specifinog reenja za otklanjanje konkretnog problema pristrasnosti. Ova tehnika primenjena je na poznati skup recenzija sa sajta Amazon, u cilju predvianja da li je ocena proizvoda pozitivna ili negativna. Ispostavlja se da obrtanje gradijenta uvek daje bolje rezultate.

1. Priprema podataka ⮭¶

Razmatrani skup podataka moe se preuzeti ovde i sadri veliki broj recenzija sa sajta Amazon. Recenzije su pretprocesirane i date u $n$-gramskoj formi (vrea unigrama i bigrama), oznaene kao pozitivne ili negativne (balansirane su po tom kriterijumu), kao i podeljene po domenu, odnosno vrsti proizvoda na koji se odnose. U nastavku je data klasa koja predstavlja jednu recenziju, uvajui njene $n$-grame (tekst), oznaku (polaritet) i domen (vrsta proizvoda).

In [1]:
# Klasa koja predstavlja jednu recenziju
class Recenzija:
    # Konstrukcija na osnovu uitane linije i domena
    def __init__(self, linija, dom):
        # Podela linije na rei
        linija = linija.strip().split()
        
        # Renik recenzije kao skup n-grama sa brojem pojavljivanja
        recnik = [(r[:r.rfind(':')], int(r[r.rfind(':')+1:])) for r in linija[:-1]]
        
        # Pretvaranje renika u niz n-grama
        tekst = [rec for rec, i in recnik for _ in range(i)]
        
        # Spajanje niza n-grama u nisku (tekst)
        self.tekst = ' '.join(tekst)
        
        # uvanje oznake (polariteta) recenzije
        self.oznaka = int(linija[-1] == '#label#:negative')
        
        # uvanje domena (vrste) proizvoda
        self.dom = dom

Pretpostavka je da unutar svakog pojedinanog domena moe postojati izvor pristrasnosti. Primera radi, korisnici mogu na specifian nain izraavati (ne)zadovoljstvo prema odreenom tipu proizvoda. Klasifikatore to ne bi trebalo da zanima, ve bi idealno bilo da budu u stanju da prepoznaju opte naine izraavanja (ne)zadovoljstva, nevezano za vrstu proizvoda. Ideja je, dakle, obuiti modele koji ne uzimaju u obzir domen prilikom klasifikacije. Za potrebe ovog eksperimenta, izabrana su dva tipa prozivoda: elektronski i kuhinjski. Konkretno, recenzije elektronskih proizvoda oznaene su kao izvorni domen, a kuhinjskih kao ciljni. Zamisao je, dakle, pomou recenzija elektronskih proizvoda nauiti klasifikator koji dobro klasifikuje i recenzije kuhinjskih. U nastavku su ovi podaci uitani. Konkretno, za svaki domen postoje tri datoteke: positive.review sa pozitivnim recenzijama, negative.review sa negativnim utiscima i unlabeled.review sa ocenama koje slue za proveru modela. Tako nema potrebe za dodatnom podelom na skupove za trening (obuku) i test (proveru).

# Dodavanje sadraja datoteke u podatake
def dodaj(datoteka, tip):
    with open(datoteka) as f:
        return [Recenzija(l, tip) for l in f.readlines()]

# Uitavanje datoteka odabranog domena
def uitaj(train, test, domen, tip):
    # Dodavanje pozitivnih recenzija
    train.extend(dodaj(f'podaci/{domen}/positive.review', tip))
    
    # Dodavanje negativnih recenzija
    train.extend(dodaj(f'podaci/{domen}/negative.review', tip))
    
    # Dodavanje test recenzija
    test.extend(dodaj(f'podaci/{domen}/unlabeled.review', tip))

import pandas as pd

# Priprema podataka za dalji rad
def pripremi(podaci, vektorizator):
    # Vrea rei prosleenog skupa
    x = vektorizator.transform([t.tekst for t in podaci])
    x = pd.DataFrame(x.todense(), columns=vektorizator.get_feature_names_out())
    
    # Oznake i domen prosleenog skupa
    y = pd.Series([t.oznaka for t in podaci])
    z = pd.Series([t.dom for t in podaci])
    
    # Vraanje pripremljenih podataka
    return x, y, z

2. Tradicionalni model ⮭¶

Prvo je formiran tradicionalni model, koji ne uzima u obzir domen instance. Konkretno, ovaj model predstavlja jednostavnu neuronsku mreu sa samo jednim potpuno povezanim skrivenim slojem (enkoderom), ija je aktivacija linearna ispravljaka jedinica (ReLU), dok je na izlazu iz mree sigmoidna aktivacija. Prvo je napisana bazna klasa, koja e biti koriena i u nastavku, a zatim konkretna klasa, koja predstavlja opisani tradicionalni neuronski klasifikator.

In [7]:
# Modul za klasini rad sa mreama
import torch, torch.nn as nn

# Osnovna klasa za sve dalje modele
class Suparnik(nn.Module):
    # Konstrukcija na osnovu arhitekturalnih parametara
    def __init__(self, ulaz=n, skriveni=4, klase=1, seme=0):
        # Inicijalizacija PyTorch modula
        super().__init__()
        
        # Fiksiranje izvora pseudosluajnosti
        torch.manual_seed(seme)
        
        # Linearna ispravljaka aktivacija
        self.relu = nn.ReLU()
        
        # Sigmoidna aktivacija
        self.sigm = nn.Sigmoid()
        
        # Potpuno povezani skriveni sloj
        self.enc = nn.Linear(ulaz, skriveni, bias=False)
        
        # Potpuno povezani izlazni sloj
        self.klas = nn.Linear(skriveni, klase, bias=False)
In [8]:
# Tradicionalni klasifikator
class Tradicionalni(Suparnik):
    # Konstrukcija bazne klase
    def __init__(self, ulaz=n, skriveni=4, klase=1, seme=0):
        super().__init__(n, skriveni, klase, seme)
    
    # Funkcija za propagaciju unapred
    def forward(self, x):
        # Enkodiranje ulaza
        x = self.enc(x)
        
        # Ispravljaka aktivacija
        x = self.relu(x)
        
        # Dekodiranje izlaza
        y = self.klas(x)
        
        # Sigmoidna aktivacija
        y = self.sigm(y)
        
        # Vraanje rezultata
        return y

Kako je za rad sa neuronskim mreama koriena biblioteka PyTorch, neophodno je runo napisati funkciju za obuavanje modela. Koristi se optimizator Adam, sa neto veim brojem epoha. Napisana je i pomona funkcija za prevod podataka u odgovarajui oblik, to su posebni PyTorch tenzori.

Kako podataka nema previe, ne dele se na pakete, ve se ceo skup za obuku izjedna provlai kroz neuronsku mreu. Za potrebe auriranja parametara mree na kraju epohe, kao funkcija greke koristi se binarna unakrsna entropija. Na kraju obuke, prikazuje se grafik toka greke kroz vreme (epohe).

In [11]:
# Obuavanje tradicionalne mree
def obui(model, x_train, y_train, epoha=epoha):
    # Pravljenje optimizatora nad mreom
    if not hasattr(model, 'adam'):
        model.adam = Adam(model.parameters())
    
    # Binarna unakrsna entropija kao greka
    if not hasattr(model, 'bce'):
        model.bce = nn.BCELoss()
    
    # Postavljanje moda za obuku
    model.train()
    
    # Niz greaka po epohama
    greke = []
    
    # Iteracija kroz epohe
    for i in range(epoha):
        # Dohvatanje podataka kao nizova
        x = x_train.to_numpy()
        y = y_train.to_numpy()
        
        # Dohvatanje podataka kao tenzora
        x = tenzor_od_niza(x)
        y = tenzor_od_niza(y).unsqueeze(1)
        
        # Propagacija unapred kroz model
        y_pred = model(x)
        
        # Izraunavanje greke predvianja
        greka = model.bce(y_pred, y)
        
        # Dodavanje greke u niz po epohama
        greke.append(greka.item())
        
        # Osveavanje stanja optimizatora
        model.adam.zero_grad()
        
        # Propagacija greke unazad
        greka.backward()
        
        # Korak optimizatora
        model.adam.step()
    
    # Postavljanje celobrojnih osa grafika
    plt.axes().xaxis.set_major_locator(MaxNLocator(integer=True))
    
    # Crtanje toka greke kroz vreme (epohe)
    plt.plot(range(1, epoha+1), greke)
    
    # Vraanje modela
    return model

# Dodavanje nove funkcije klasi
Tradicionalni.obui = obui

Sada je mogue obuiti tradicionalni model. Konkretno, obuavaju se tri modela: prvi koji ui samo na recenzijama elektronskih ureaja (izvorni domen), drugi koji ui samo na recenzijama kuhinjskih ureaja (ciljni domen) i trei koji ui na svim recenzijama iz skupa za obuku, dakle na obe vrste ureaja.

3. Obrtanje gradijenta ⮭¶

Detalji o obrtanju gradijenta kao ideji mogu se videti ovde, dok se njegova primena u klasifikaciji teksta moe videti ovde. Ukratko, ova tehnika se odlikuje grananjem neuronske mree, pri emu postoji zajedniki deo, koji se zatim deli na dve grane glavnu i suparniku. Glavna grana reava konkretan problem (klasifikacija), dok suparnika otkriva eventualni izvor pristrasnosti (domen). Prilikom auriranja parametara modela, zasnovanog na gradijentnoj minimizaciji greke, gradijent se obre (negira) na zajednikom delu suparnike grane. Ovime se efektivno otklanja izvor pristrasnosti, pa je rezultujui model (predstaven glavnom granom) robusniji. U nastavku je implementirana funkcija koja obre gradijent, kao i opisani model sa grananjem za potrebe rada sa recenzijama.

In [16]:
# Modul za rad sa funkcijama
from torch.autograd import Function

# Funkcija za obrtanje gradijenta
class Obrtanje(Function):
    # Propagacija unapred je neizmenjena
    @staticmethod
    def forward(ctx, x):
        return x
    
    # Propagacija unazad negira gradijent
    @staticmethod
    def backward(ctx, grad):
        return -grad

# uvanje funkcije za laku upotrebu
obrni = Obrtanje.apply

class Domenski(Suparnik):

Jo jednom je neophodno runo napisati funkciju za obuku. Ovog puta ima malih izmena. Naime, razgranati suparniki model mogue je trenirati na dva naina. Podrazumevano, samo se instance iz izvornog domena koriste u optimizaciji parametara klasifikacione grane, dok se svi podaci koriste pri obuci suparnike grane. Alternativno, svi parametri obuavaju se pomou svih podataka. Implementirana su i primenjena oba naina, dakle dva nova modela. Oekivano je i primetno da je suparnika grana donekle nestabilna. Njen cilj je, naime, da to bolje klasifikuje domen, ali je to samo usputna stvar. Glavni cilj biva ispunjen zbog obrtanja gradijenta, a to je da skrivena reprezentacija zapravo u sebi ne skladiti podatke o domenu, ve samo neke opte informacije o polaritetu recenzije. Kako postoje suprotstavljeni ciljevi, nestabilnost nije neobina. Vano je da je na kraju obuke greka u relativno stabilnom padu.

In [19]:
# Obuavanje suparnike mree
def obui(model, x_train, y_train, z_train, svi, epoha=epoha):
    # Pravljenje optimizatora nad mreom
    if not hasattr(model, 'adam'):
        model.adam = Adam(model.parameters())
    
    # Binarna unakrsna entropija kao greka
    if not hasattr(model, 'bce'):
        model.bce = nn.BCELoss()
    
    # Postavljanje moda za obuku
    model.train()
    
    # Niz greaka po epohama
    gklas = []
    gdom = []
    greke = []
    
    # Iteracija kroz epohe
    for i in range(1, epoha+1):
        # Dohvatanje podataka kao nizova
        x = x_train.to_numpy()
        y = y_train.to_numpy()
        z = z_train.to_numpy()
        
        # Dohvatanje podataka kao tenzora
        x = tenzor_od_niza(x)
        y = tenzor_od_niza(y).unsqueeze(1)
        z = tenzor_od_niza(z).unsqueeze(1)
        
        # Propagacija unapred kroz model
        try:
            y_pred, z_pred = model(x, y)
        except:
            y_pred, z_pred = model(x)
        
        # Eventualno izdvajanje izvornog domena
        if not svi:
            neki = torch.tensor(np.flatnonzero(z_train))
            y_pred = torch.index_select(y_pred, 0, neki)
            y = torch.index_select(y, 0, neki)
        
        # Izraunavanje greke predvianja oznake
        g1 = model.bce(y_pred, y)
        gklas.append(g1.item())
        
        # Izraunavanje greke predvianja domena
        g2 = model.bce(z_pred, z)
        gdom.append(g2.item())
        
        # Zbirna greka predvianja
        g = g1 + g2
        greke.append(g.item())
        
        # Osveavanje stanja optimizatora
        model.adam.zero_grad()
        
        # Propagacija greke unazad
        g.backward()
        
        # Korak optimizatora
        model.adam.step()
    
    # Postavljanje celobrojnih osa grafika
    plt.axes().xaxis.set_major_locator(MaxNLocator(integer=True))
    
    # Crtanje toka greaka kroz vreme (epohe)
    plt.plot(range(1, epoha+1), gklas, label='class')
    plt.plot(range(1, epoha+1), gdom, label='advers')
    plt.plot(range(1, epoha+1), greke, label='total')
    
    # Ucrtavanje legende grafika
    plt.legend()
    
    # Vraanje modela
    return model

# Dodavanje nove funkcije klasi
Suparnik.obui = obui

Alternativno, problemu se moe pristupiti na drugi nain, iji se detalji mogu videti ovde. U ovom pristupu, nema eksplicitnog grananja modela, ve se suparnika grana nadovezuje na rezultat sigmoidno dekodiranog izlaza klasifikatora. Konkretno, formira se vektor od tri vrednosti, koje na razliite naine oslikavaju korektnost predvianja, a zatim se ta reprezentacija dekodira kako bi se predvideo domen. Oigledno, na ovaj nain nije mogue predvideti domen neobeleenih instanci, mada to i nije neophodno. S druge strane, obuka je stabilna. Ovaj pristup je popularan u obradi prirodnih jezika, budui da prua jednakost mogunosti, koja je neophodna u izgradnji nepristrasnih jezikih modela. I ovaj model se, potpuno isto kao i prethodni, obuava na dva naina.

# Nepristrasni klasifikator
class Nepristrastan(Suparnik):
    # Konstrukcija bazne klase:
    def __init__(self, ulaz=n, skriveni=4, klase=1, seme=0):
        super().__init__(n, skriveni, klase, seme)
        
        # Skala sigurnosti klasifikatora
        self.c = nn.Linear(klase, klase, bias=False)
        
        # Potpuno povezani domenski sloj
        self.deb = nn.Linear(3, klase)
    
    # Funkcija za propagaciju unapred
    def forward(self, x, yt=None):
        # Enkodiranje ulaza
        x = self.enc(x)
        
        # Ispravljaka aktivacija
        x = self.relu(x)
        
        # Dekodiranje izlaza
        y = self.klas(x)
        
        # Skalirana sigmoidna aktivacija
        y = self.sigm(self.c(y))
        
        # Kraj ukoliko nije poznata klasa
        if yt is None:
            return y
        
        # Obrtanje gradijenta
        z = obrni(y)
        
        # Meureprezentacija predvianja
        z = torch.cat([z, z * yt, z * (1 - yt)], dim=1)
        
        # Dekodiranje domena
        z = self.deb(z)
        
        # Sigmoidna aktivacija
        z = self.sigm(z)
        
        # Vraanje rezultata
        return y, z

4. Evaluacija modela ⮭¶

Za kraj, modeli se evaluiraju, kako bi se mogli uporediti. Prvo je implementiran metod za sprovoenje klasifikacije, a zatim i metod za evaluaciju. Kako se razmatra problem binarne klasifikacije, drugi metod prikazuje matricu konfuzije, koja razdvaja tano klasifikovane recenzije od netano kategorizovanih. Matrica je bolja utoliko to vie odgovara dijagonalnoj. Prikazana je i procentualna mera kvaliteta $F_1$, koja je direktno srazmerna kvalitetu klasifikacije.

In [26]:
# Klasifikacija odabarnim modelom
def predvidi(model, x_test):
    # Postavljanje moda za proveru
    model.eval()
    
    # Zanemarivanje gradijenata
    with torch.no_grad():
        # Dohvatanje podataka kao nizova
        x = x_test.to_numpy()
        
        # Dohvatanje podataka kao tenzora
        x = tenzor_od_niza(x)
        
        # Propagacija unapred kroz model
        y_pred = model(x)
        
        # Dohvatanje klase ako nije jedina
        if isinstance(y_pred, tuple):
            y_pred = y_pred[0]
        
        # Zaokruivanje predvianja
        return y_pred.numpy()[:, -1] > .5

# Dodavanje nove funkcije klasi
Suparnik.predvidi = predvidi

from sklearn.metrics import f1_score

# Modul za rad sa matricama konfuzije
from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay

# Pravljenje jedne matrice konfuzije
def matrica(model, x_test, y_test, domen, ax):
    # Predvianje modela na skupu za proveru
    y_pred = model.predvidi(x_test)
    
    # Matrica konfuzije na osnovu predvianja
    mat = confusion_matrix(y_test, y_pred)
    
    # Crtanje izraunate matrice konfuzije
    ConfusionMatrixDisplay(mat).plot(cmap='binary', colorbar=False, ax=ax)
    
    # F1 mera kvaliteta na osnovu predvianja
    f1 = f1_score(y_test, y_pred)
    
    # Upisivanje domena i mere kvaliteta
    ax.set_title(f'{domen} [F$_1$ = {f1:.2%}]')

# Funkcija za crtanje matrica konfuzije
def konfuzija(model, x_test, y_test, z_test):
    # Podela grafika na dva dela
    fig, axs = plt.subplots(1, 2, figsize=(8, 4), sharex=True, sharey=True)
    
    # Matrica konfuzije izvornog domena
    matrica(model, x_test[z_test], y_test[z_test], 'Elektronika', axs[0])
    
    # Matrica konfuzije ciljnog domena
    matrica(model, x_test[~z_test], y_test[~z_test], 'Kuhinja', axs[1])

# Dodavanje nove funkcije klasi
Suparnik.konf = konfuzija

Razgranati domenski model, meutim, donosi osetno poboljanje na oba domena, pa je sada kvalitet oko $88\%$.
